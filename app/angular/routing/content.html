Angular Routing

Routing in Angular isn't handled by the core framework in Angular 1.x. Thus, there are several competing libraries that are used for routing. The core team created ngRoute for simple routing needs. For most larger applications, the currently preferred library is ui-router. UI Router uses states instead of routes which allows for more complex behavior such as named, nested and parallel views.

To use UI Router, the ui.router module must be added to your application's dependencies. 

// myrouter-module.js
// Create module
angular.module('myRoutingModule', ['ui.router']);
// mymodule-router.js
// Retreive router moduleand configure states
angular.module('myRoutingModule')
.config(['$stateProvider', '$urlRouterProvider',
  function($stateProvider, $urlRouterProvider) {
    'use strict';
  
    // For any unmatched url, redirect to /home
    $urlRouterProvider.otherwise("/home");
  
    $stateProvider
      .state('home', {
        url: "/home",
        templateUrl: "home.html"
      })
      .state('about', {
        url: "/about",
        templateUrl: "about.html",
        controller: 'myAboutController',
        controllerAs: 'myAboutController'
      })
  }
]);

//index.html

Create a separate module for the router in your Angular application. Only require ui.router on that module instead of on the global module. This will help isolate the routing code to enable easier migration to a different routing library in the future if needed.
